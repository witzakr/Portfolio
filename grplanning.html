The group project is divided into 4 sprints. The first one is responsible for the introduction, which will be a project plan, introducing group members to the subject and overall establishing common rules. The second sprint is responsible mostly for research and communication because during this phase students will be responsible for gathering data to create user requirements and functions, which will used later to create diagrams, which will be crucial in the later stage of the project. In order to get that data, the group members will have to keep in touch with a client and conduct surveys and interviews with the specified target for the website. The third sprint is the ideation phase, which will focus on ideation techniques, such as brainstorming, brainwriting, worst possible ideas, and others. Before the end of this sprint, group members will have to create a record project pitch presentation, in order to present their ideas and progress. The last sprint, which is the fourth one, is related to the development process. In this stage, group members will create a prototype of the website and after that will move on to the testing phase.
